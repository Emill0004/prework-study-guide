HTML

  • Every element has angle brackets on both sides.
  • !DOCTYPE html - tells the browser that this is an HTML document.
  • The html opening element represents the start of the html document.
    • lang="en" - tells the browser the language for the website, en meaning english.

  • head - contains information about the webpage (not seen by the user).
    • title - sets the title of the webpage.
    • style - defines the CSS styles associated with the website.
    • base - defines the base URL for a website.
    • script - embeds client-side scripts in the HTML document.
    • link - connects an external resource to the HTML document.
    • meta - helps the browser render the page correctly.
      • charset - sets character encoding.
      • http-equiv - sets an HTTP header for the content.
      • viewport - sets the default view for the webpage.

  • body - represents the visible content shown to the user.
    • h1 through h6 - represent different levels of headers. h1 being the largest and h6 being the smallest.
    • p - represents a paragraph.
    • ul - unordered list (bullet points).
    • ol - ordered list (numbered list).
    • li - list item.
    • img - contains information about an image being displayed on the website.
      • src - defines the location of the image.
      • alt - text to describe the image and it's purpose.
    • br - a line of empty space (line break).
    • div - creates a division in the page.
    • span - creates an inline division in the page.
    • a - anchor, used to create a link to a webpage.
      • href - points to the URL for the link.
    • section - a semantic element that defines sections in the document.
    • footer - usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • Inline CSS - applies to a specific HTML tag using a style attribute with a CSS rule.
  • Internal style sheet:
    • Uses the style element in the head of the document.
    • Configures CSS classes and IDs (styles all indicated elements in the page).
  • External CSS style Sheet:
    • Allows developers to keep all their CSS rules in a separate file (makes design changes easier).
    • uses the link element in the head of the document.
  • margin - a CSS property that indicates how much space is around the outside of an element.
    • a margin value of 0 means there is no extra space around the border of the element.
  • padding - adds space around the content inside the element.
    • Increasing padding makes the element itself bigger.
    • (Margin(Border(Padding(Content))))
  • * - a selector that includes all elements for the CSS declarations.
  • text-align - aligns text (e.g. text-align: center;).
  • box-shadow - adds shadow effects around an element’s frame.
  • display - a declaration related to images.
    • block - a display value which makes each image start on a new line.

  • Linking an external CSS style sheet:
    • (in angle brackets) link rel=”stylesheet” href=”path to .css document”
    • rel - specifies the relationship between the current document and the linked resource.
    • href - specifies the location (URL) of the external resource IT NEEDS THE PATH NOT JUST THE URL.

  • Class attribute - enables you to choose specific elements to apply CSS rules to.
    • You need to declare that a given element is a certain class “class=”class name here”.
    • In the selector on the .css document there needs to be a . before the name of the class.

Git

  • pwd - displays the current working directory.
  • ls - displays the list of files and folders in the current directory.
  • cd - navigates between directories in the path.
  • cd .. - navigates backwards one level at a time, through the path of your directories.
  • mkdir - creates a new directory.
  • touch - creates a file inside a directory.
  • start - opens a file.
  • git add -A - commits all files in the working branch.
  • git commit -m - commits staged files to the repository and links a message to it.
  • git pull origin [base branch name] - pulls in the base branch.
  • git push origin [new branch name] - pushes all of the changes from the new branch up.
  • code . - git bash command to open VS Code.
  • Repo - a folder for your project, contains all of the files and version histories.
  • Branch - a workspace where you can make changes that don't affect the live site.
  • Markdown (.md) - the way content files are formatted in github, (uses text that easily converts to HTML).
  • Commit changes - adds changes to your branch (adding, editing, or deleting a file) saves a new version of the file
  • you're working on.
  • Pull request (“PR”) - asks to merge changes from one branch into another.
  • Merge - “pulls in” changes after a PR is approved (changes apply to the live site).
  • Issue - used to report new tasks/content fixes, tracked on the project board.
  • Federalist - a platform that securely deploys a website from a github repository (lets users preview proposed changes).

  • Creating a github repository:
    1. Click the + icon in the top right corner of github.com and then the new repository.
    2. Name the repository and write a description.
    3. Add a README file.
    4. Select the .gitignore template: Node.
    5. Select the MIT License.
    6. Click create repository.
    7. Click the ‘<> Code’ dropdown menu.
    8. Select the SSH tab and copy the url.
    9. Navigate to the desired file location in git bash.
    10. Type ‘git clone [copied SSH link].

  • Each commit should contain an isolated complete change.
  • Committing and pushing changes backs up your work to remote storage.
  • Make a separate branch for each set of unrelated changes.
  • Link addressed issues with each pull request to make it easier for reviewers.
  • As a reviewer you should leave questions, comments, and suggestions (you can comment on the whole pr or individual lines).
  • Delete your branch after it has been merged (doesn’t delete pull request and commit history).
  • Feature branch - copy of the main branch, where the new code is added.
  • Default branch is called either ‘main’ or ‘master’ (dont code directly in the main branch).
  • Be sure to commit often.
  • Base branch - where the feature branch’s changes will be applied (can be the main branch).

JavaScript

  • APIs - browser application programming interfaces.
    • Dynamically creates HTML and sets CSS styles.
    • Collects and manipulates the video stream from a usere’s webcam.
    • Generates 3D graphics and audio samples.
    • Third-party APIs allow developers to incorporate functionality from other content providers.

  • Third-party frameworks and libraries can be applied to HTML to accelerate the process of building sites and applications.

  • The script element links the .js file to the HTML file.
    • Works just like the link element.
    • Goes at the end of the body section because it's important to have all the HTML loaded before you run JavaScript that could affect it.

  • Variable - container that stores a value.
    • You can declare a variable with the let or var keyword (let myVariable;).
    • You can give variables a value after its been declared (myVariable = “Bob”;)
    • You can declare a variable and assign it a value in the same line (let myVariable = “Bob”;).
    • You can retrieve the value by calling the variable name (myVariable;).
    • You can change a variable’s value later in the code.
    • String - a sequence of text.
      • myVariable = ‘Bob’; or myVariable = “Bob”;
    • Number - it's a number.
      • myVariable = 10;
    • Boolean - True/False value.
      • myVariable = true; or myVariable = false;
    • Array - a structure that lets you store multiple values in a single reference.
      • myVariable = [1,'Bob','Steve',10];
      • Refer to each value in the array like this: myVariable[0], myVariable[1] etc.
      • Has a property called .length, which is equal to the number of items in the array.
    • Object - anything, everything in javascript is an object and can be stored in a variable.
      • myVariable = document.querySelector(‘h1);
      • All of the other examples too.
    • Undefined - represents the absence of a value.

  • Semicolon - indicates where a statement ends.
  • JavaScript is case sensitive, meaning myVariable is not the same as myvariable.
  • Comments - snippets of text that the browser ignores, there for developers to read.
    • /* - starts the commented lines.
    • */ - stops the commented lines.
    • // - turns a single line into a comment.

  • Operator - a mathematical symbol that produces a result based on two values.
    • + - adds numbers or combines strings.
    • - - subtracts two numbers.
    • * - multiplies two numbers.
    • / - divides two numbers.
    • = - assigns a value to a variable.
    • === - tests to see if two values are equal, returns a boolean result.
    • !, !== - tests to see if the variable is not equal to a given value.

  • Conditionals - code structures used to test if an expression returns true or not (if else).
  • Functions - a way of packaging functionality that can be reused.
    • Arguments - data needed for the function to execute the code properly, arguments go inside the () when a function is called.
    • document.querySelector(“”) and alert(“”) are functions that are built into the browser.
    • Return - a statement that tells the browser to return the stated variable so we can use it outside of the function.

  • Events - code structures that listen for activity in the browser, and run code in response.
    • Click event - an event that's initiated by the user clicking on something.

  • Primitive values - immutable values represented directly at the lowest level of the language.
  • Control flow - the order in which a computer executes code in a script (code runs from the top down in JavaScript).
  • Truthy and falsy - if statements check for truthy or falsy statements.
    • Falsy - 0, -0, On, “”, null, undefined, NaN, false boolean.
    • Truthy - anything that isn't falsy.

  • Else if - used to provide more conditions in an if statement.
  • x++ - used in a for loop to represent that for each step x is equal to the current value of x plus 1.
  • Math.floor() and Math.random() can be used to generate random numbers.
